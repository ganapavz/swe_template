Assets

Templates

The QGov assets contain a number of server side includes as well as an XHTML skeleton.  In order to have the same templating functionality in JSF, a number of facelets templates were created that closely align with the QGov includes and skeleton.


CBUI Template Related QGov asset Description
basicPageTemplate.xhtml

xhtml-skeleton.html

breadcrumb-global.html

This template closely aligns with the QGov XHTML skeleton, with JSF ui:includes/ui:inserts in place of SSI. It does not contain any TMR authored composite components (it is intended to be decoupled, and to be as close as possible to the QGov XHTML skeleton).
documentProperties.xhtml xhtml-skelton.html JSF template to abstract the document properties from the XHTML skeleton.
footer.xhtml footer.html JSF template version of SSI

JSF template version of SSI (JSF markup used for hrefs in stylesheets, script tags, and conditional comments for IE).

URLs for CSS and javascript modified to internally packaged JSF URL (not QGov /assets URL).

header.xhtml header.html JSF template version of SSI (JSF markup used for images and to output conditional comments for IE)
metadata.xhtml xhtml-skeleton.html This template contains the metadata originally in the XHTML skeleton (just to make the page template tidier)

Javascript

Only a few scripts were copied, most with some modifications to get the to work with JSF.

In order for images/javascript to be loaded as JSF resources, the following javascript had to be added to the templates: 

<script type="text/javascript">
    qg.swe.paths.assets = '#{request.contextPath}/javax.faces.resource/';
    qg.swe.paths.assetsSuffix = '.xhtml?ln=qgov';
</script>

and the javascript updated to append the suffix.

loader.js Modified javascript to add qg.swe.paths.assetsSuffix to asset URLs
qgov-environment.js None

Integration Issues

There were a number of modifications required to get the QGov assets working with JSF and Primefaces and there's a number of outstanding issues.

  • Analytics (Google analytics and SiteCatalyst). We haven't included it yet, but we do want GA. Ben Boyle has said we can get access to the reports (but he'll need to update the javascript so it works for our domains).
  • Release cycle - maintenance of templates, and consumption of new assets (what's the best way forward?)
  • Page metadata - don't currently use it, but if we do what are the benefits and what to we put in there?
  • Styles conflicting with primefaces (in particular the 'form layout' ones) - we've just removed them from qgov.css, but this is not really ideal.
  • Weird bouncing when trying to scroll to bottom of page in IE.
  • Consider packaging unminimized js/css for debugging in dev environment?
  • Really hard to find stuff on Govdex!
